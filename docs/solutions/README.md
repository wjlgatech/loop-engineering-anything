# docs/solutions — decision records

Short, durable records of *why* a non-obvious engineering decision was made —
the reasoning that would otherwise live only in a plan or a CHANGELOG line and
get lost. Distinct from `docs/plans/` (narrative, forward-looking) and
`CHANGELOG.md` (what changed): a solution doc captures a decision and the
measurement or constraint that settled it, so it is not re-litigated.

One file per decision, kebab-case. Keep them short.

# marketing/ — the promotion kit

Promotional assets for `loop-engineering-anything`, plus the **reusable pattern**
behind them. We run promotion as a loop (the same thesis the repo is built on):
draft → grade against a checklist → revise → ship → record the learning.

## Files

| File | What it is |
|---|---|
| [`PATTERN.md`](PATTERN.md) | **Start here.** The reusable dev-tool launch playbook: the reverse-engineered pattern, why it works, our 10× moves, a fill-in template, a pre-publish grader checklist, the fact base, and how to run a launch as a loop. |
| [`linkedin-launch.md`](linkedin-launch.md) | The LinkedIn post (full + short variants + first comment). |
| [`x-thread.md`](x-thread.md) | The X/Twitter launch thread. |
| [`show-hn.md`](show-hn.md) | Show HN title + technical first comment (honest about maturity). |
| [`taglines.md`](taglines.md) | Reusable hooks, one-liners, and proof points. |
| [`dogfood-log.md`](dogfood-log.md) | The optimization log — one entry per launch, with hypothesis → result → learning. |
| [`content/`](content/) | **Dated, ready-to-post instances** produced from the kit (e.g. `2026-06-16-longform-*`, `2026-06-16-short-a/b-*`). Templates above are reusable; these are the generated copies, each with a `date` + `channel` in frontmatter. |

## How to use it

1. Read `PATTERN.md`.
2. Refresh the **fact base** (`PATTERN.md` §6) against the current repo so every
   number is true.
3. Open the asset for your channel, draft from the template, **grade against the
   checklist** (`PATTERN.md` §4), revise until it passes.
4. Ship it, then add a `dogfood-log.md` entry (hypothesis + result + learning) so
   the next launch compounds.

> Everything here must be defensible against the repo. Market the proven core
> (the convergence loop, reality-grounded judging, safety, compounding); never
> imply deferred/frontier work is already shipped.

**1/**
Stop generating tools. Start engineering loops.

Your AI writes a CLI once and freezes it. Nothing closes the gap between "it exists" and "it's actually good."

We open-sourced loop-engineering-anything to close it 🧵

**2/**
Point it at any API or codebase.

It builds an agent-native CLI, grades it against reality, and refactors until the grade stops climbing — then reports how it got there.

Runs locally. Your code never leaves your machine.

**3/**
The trick isn't a smarter model. It's the loop:

route → generate → judge → refactor → re-judge → compound

**4/**
The judge is independent.

5 dimensions, A–F, graded against real captured payloads — never the model grading its own code.

Quality comes from reality, not vibes.

**5/**
It can't degrade into a worse tool.

It stops at the first of: target grade · plateau · budget · safety block.

A safety failure is terminal — an unsafe tool never ships, no matter how high it scores.

**6/**
It compounds.

Every accepted fix becomes a regression test, so a solved problem never comes back. The system gets harder to break the longer it runs.

**7/**
And "converged" is a claim, not a fact, until a human confirms it. Maker ≠ checker is enforced.

(We even run this launch as a loop we keep grading. Dogfood, all the way down.)

**8/**
Everyone's optimizing prompts. The next leverage is the loop.
